## 필요한 것들

Vercel 배포를 시작하기 전에 아래 항목을 준비하세요.

### 1. GitHub 계정

Vercel은 GitHub, GitLab, Bitbucket과 연동됩니다. 이 가이드에서는 **GitHub**을 사용합니다.

- [github.com](https://github.com) 에서 계정 생성
- 이미 계정이 있다면 그대로 사용

### 2. Node.js 설치

Next.js를 실행하려면 Node.js가 필요합니다.

```bash
# Node.js 버전 확인
node -v

# 18.18.0 이상이면 OK
```

<Callout type="tip">
Node.js가 없다면 [nodejs.org](https://nodejs.org)에서 LTS 버전을 설치하세요.
nvm을 사용하면 버전 관리가 편리합니다.
</Callout>

### 3. 패키지 매니저

이 가이드에서는 **pnpm**을 사용합니다.

```bash
# pnpm 설치
npm install -g pnpm

# 설치 확인
pnpm -v
```

npm이나 yarn을 사용해도 무방합니다.

### 4. Vercel 계정

1. [vercel.com](https://vercel.com)에 접속
2. **Continue with GitHub** 클릭
3. GitHub 계정으로 로그인
4. 권한 승인

<Callout type="info">
Vercel 무료 플랜(Hobby)으로 개인 프로젝트를 충분히 배포할 수 있습니다.
월 100GB 대역폭, 빌드 100시간이 제공됩니다.
</Callout>

## 체크리스트

준비가 완료되었는지 확인하세요:

- GitHub 계정 로그인 가능
- `node -v` 실행 시 18.18.0 이상 출력
- `pnpm -v` 실행 시 버전 출력
- Vercel 대시보드 접속 가능
